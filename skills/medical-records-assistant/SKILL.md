# 🏥 医疗档案助手

**MRCE 档案管理原则**: Medical Record Classification by Event
- **M**onitor (监测): 健康看板快速发现问题
- **R**ecord (记录): 时间轴追踪变化趋势  
- **C**lassify (分类): 按器官/病种专项管理
- **E**vidence (证据): 原始报告完整归档

## 🔄 标准工作流程

### 场景1: 收到新体检报告
```
1. 用户上传报告照片/PDF
2. AI提取数据 → 生成Markdown表格
3. 对比历史数据 → 标注趋势 (↑/↓/→)
4. 识别异常项 → 按风险分级 (🔴🟡🟢)
5. 建议写入：
   - 体检档案-[姓名].md (核心看板更新)
   - [姓名]/专项病症档案/相关病种.md (专项更新)
   - Tracking/体检指标.csv (数据归档)
6. 原始文件 → 存入 [姓名]/报告原件/
7. 生成复查建议/待办清单
```

### 场景2: 发现新病症/异常指标
```
1. AI识别需要长期跟踪的病症
2. 建议创建专项病症档案：
   [姓名]/专项病症档案/[病种]专项档案.md
3. 档案结构：
   - 关键指标趋势表（时间倒序）
   - 详细检查记录（含报告链接）
   - 医生建议汇总
   - 决策备忘录（手术临界点等）
4. 更新健康事件记录.csv
5. 核心看板标注风险等级
```

### 场景3: 就医全流程
```
就医前：
1. 告诉AI "我要看XX科"
2. AI读取历史档案 → 生成：
   - 病情时间线摘要
   - 关键指标趋势图
   - 建议追问问题清单

就医后：
1. 提供医生结论 → AI更新档案
2. 新增检查 → 按场景1处理
3. 调整用药 → 更新就医记录.md
4. 设置复查提醒
```

### 场景4: 季度健康复盘
```
1. 导出Apple Watch数据 → AI分析趋势
2. 对比近期体检数据 → 综合分析
3. 更新各专项病症档案趋势表
4. 生成个性化日常管理方案
5. 检查复查计划 → 更新日历
```

## 🏷️ 状态标注规范

| 图标 | 含义 | 使用场景 |
|------|------|----------|
| 🟢 | 低风险/稳定 | 指标正常或多年稳定 |
| 🟡 | 中风险/关注 | 偏高偏低或需定期监测 |
| 🔴 | 高风险/紧急 | 严重异常或快速增长 |
| ✅ | 完成/正常 | 复查完成或指标正常 |
| ⚠️ | 需关注 | 边界值或新发现异常 |
| ⬆️ | 升高 | 相比上次上升 |
| ⬇️ | 降低 | 相比上次下降 |
| → | 稳定 | 与上次持平 |
