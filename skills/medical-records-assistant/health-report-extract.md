# 体检/检查报告数据提取

当用户提供体检报告、检查报告、病理报告、影像报告的照片或PDF时，按以下流程处理。

---

## 📋 提取内容

### 1. 基本信息
- 患者姓名（建议用代号）
- 检查日期
- 医院/体检机构
- 检查类型（超声/CT/MRI/胃镜/肠镜/血液检查等）
- 科室/医师

### 2. 检查项目详情（每项必须包含）
| 字段 | 说明 |
|------|------|
| 项目名称 | 标准医学术语 |
| 检查结果 | 具体数值或描述 |
| 参考范围 | 正常范围 |
| 状态 | ✅正常 / ⚠️偏高 / ⚠️偏低 / 🔴异常 |
| 趋势 | 对比历史：↑升高 / ↓降低 / →稳定 / ❓首次 |

---

## 🗂️ 档案更新建议

根据MRCE原则，建议用户将数据更新到以下位置：

### 1. 核心看板更新
- **文件**：`体检档案-[姓名].md`
- **更新内容**：
  - 状态概览表（新增/修改相关病症行）
  - 待办清单（新增复查计划）

### 2. 专项病症档案更新
如发现需要长期跟踪的病症：
- **文件**：`[姓名]/专项病症档案/[病种]专项档案.md`
- **操作**：
  - 更新「关键指标趋势」表（新增一行）
  - 添加「详细检查记录」（复制模板填写）
  - 更新「当前状态摘要」
  - 修改风险等级（如需要）

- **新发病症**：
  - 建议创建新的专项档案：`[病种]专项档案.md`
  - 更新核心看板的「状态概览」表（新增行）

### 3. 健康事件记录更新
- **文件**：`[姓名]/专项病症档案/健康事件记录.csv`
- **添加一行**：日期、检查类型、医院、关键发现、风险等级等

### 4. 原始报告归档
- **位置**：`[姓名]/报告原件/`
- **命名建议**：`[检查类型]_[日期]_[医院].[扩展名]`
  - 示例：`超声_肝胆_2026-03-14_浙二.pdf`
  - 示例：`胃镜_2026-03-21.jpg`
  - 示例：`CT_胸部_2022-12-05.png`

## ⚠️ 注意事项

1. **完整性**：多页报告确保不遗漏任何项目
2. **准确性**：字迹不清标注"需核实"并提醒用户确认
3. **关键指标**：特别留意血糖、血脂、肝肾功能、肿瘤标志物等
4. **趋势对比**：如有历史数据，必须做趋势分析
5. **不做诊断**：只做数据解读，始终提醒用户遵医嘱
6. **风险分级**：
   - 🔴 高风险：严重异常或快速增长，需立即就医
   - 🟡 中风险：轻度异常，需定期复查
   - 🟢 低风险：稳定或多年无变化
